Forms

This page depicts all the form variants for use in this website.

Form Layouts

Three tyes of form layouts are available:

  • Vertical form (default)
  • Horizontal form
  • Inline form

Standard rules for all three form layouts:

  • Always use <form role="form"> (helps improve accessibility for people using screen readers)
  • Wrap labels and form controls in <div class="form-group"> (needed for optimum spacing)
  • Add class <form-control> to all textual <input>, <textarea>, and <select> elements so they have a width of 100%.

Vertical Form (default)

Inline Form

In an inline form, all of the elements are inline, left-aligned, and the labels are alongside.

Note: This only applies to forms within viewports that are at least 600px wide!

Additional rule for an inline form:

  • Add class .form-inline to the <form> element.

Horizontal Form

A horizontal form stands apart from the other forms both in the amount of markup, and in the presentation of the form.

Additional rules for a horizontal form:

  • Add class .form-horizontal to the <form> element.
  • Add class .control-label to all <label> elements.
  • Wrap all <input> elements with a div with the class of .input
  • Align anything to the input elements by wrapping it with the .input-pull-right class.
